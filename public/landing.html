Install

Three ways to get started

npm, npx, or directly inside Claude Code. No account required.

Global install (recommended)
Install once, use from any directory.
# Install
npm install -g ticketlens

# Connect your Jira instance
ticketlens init

# Fetch a ticket brief
ticketlens CNV1-2
Prerequisites: Node.js ≥20
Claude Code /jtb skill
Loads full ticket context directly into your Claude session.
# Install + configure
npm install -g ticketlens && ticketlens init

# Install /jtb Claude Code command
cp $(npm root -g)/ticketlens/skills/jtb/SKILL.md \
   ~/.claude/commands/jtb.md

# Then in Claude Code:
/jtb CNV1-2
Requires: Claude Code + Node.js ≥20

Yes. TicketLens supports Jira Cloud (API token), Server/DC (Bearer PAT), and older Server/DC versions that require username + password Basic auth (pre-8.14). Auth type is auto-detected from your Jira URL during ticketlens init.
Yes. The CLI produces plain text output you can pipe anywhere: ticketlens CNV1-2 --plain | pbcopy, pipe to llm, write to a file, or use in any terminal workflow. Claude Code's /jtb is an optional enhancement.
TicketLens auto-routes based on: (1) --profile=NAME flag, (2) ticket prefix match (e.g. ACME-123 → acme profile), (3) project path match (e.g. running triage in ~/projects/myteam → myteam profile), then falls back to your default profile or env vars.
It extracts requirements from the ticket (acceptance criteria, description, subtasks), reads your local VCS diff, and maps each requirement to code changes — flagging what's implemented, partially addressed, or missing. Free tier: 3 checks/month. Pro: unlimited.
Brief JSON at ~/.ticketlens/cache/PROFILE/TICKET-KEY/brief.json (4h TTL by default), and attachments at ~/.ticketlens/cache/TICKET-KEY/ (10 MB per-file cap). Pro users can set TTL to 4h, 1d, 7d, 30d, or 0 (disable) via ticketlens config.
